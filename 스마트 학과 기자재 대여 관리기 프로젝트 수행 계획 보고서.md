# 스마트 학과 기자재 대여 관리기 프로젝트 수행 계획 보고서

## 1. 프로젝트 개요 및 기술 스택 선정 (보완)

본 프로젝트는 학과 사무실에서 관리하는 노트북, 카메라 등 기자재의 대여 상태를 실시간으로 관리하는 시스템을 구축하는 것을 목표로 합니다. 제안해주신 기술 스택은 현대적인 웹 개발 트렌드에 부합하며, 특히 관계형 데이터베이스와 ORM의 조합은 복잡한 대여 이력 관리에 적합합니다.

### 1.1. 기술 스택 상세 및 추가 제안

| 구분 | 선정 기술 | 선정 이유 및 보완 사항 |
| :--- | :--- | :--- |
| **프론트엔드** | React.js | 반응형 웹 구현 및 모바일 QR 스캔 최적화에 적합합니다. **추가 제안:** 모바일 환경에서의 접근성을 높이기 위해 PWA(Progressive Web App) 도입을 고려할 수 있습니다. 또한, 서버 상태 관리를 위해 React Query(TanStack Query) 사용을 권장합니다. |
| **백엔드** | Node.js (Express) | 비동기 I/O 처리에 강점이 있어 다수의 동시 요청 처리에 유리합니다. **추가 제안:** API 문서화의 효율성을 위해 Swagger(OpenAPI)를 도입하여 프론트엔드와의 협업을 원활하게 할 수 있습니다. |
| **데이터베이스** | PostgreSQL | 복잡한 관계 처리 및 JSON 데이터 활용에 최적화되어 있습니다. 대여 이력, 기자재 상태 등 정형화된 데이터 관리에 필수적입니다. |
| **ORM** | Prisma | 직관적인 스키마 정의와 타입 안정성을 제공하여 개발 생산성을 높입니다. **추가 제안:** 동시성 제어를 위해 Prisma의 `interactive transactions` 기능을 적극 활용해야 합니다. |
| **협업 및 관리** | Git(GitHub), Figma, Postman | **추가 제안:** GitHub Actions를 활용한 CI/CD 파이프라인을 구축하여 자동화된 테스트 및 배포 환경을 마련하는 것이 좋습니다. |

## 2. 4인 팀 역할 세부 분담 (보완)

기존에 제안해주신 역할 분담은 매우 합리적입니다. 이를 바탕으로 각 파트의 전문성을 더욱 살리고, 병목 현상을 방지할 수 있도록 역할을 세분화 및 구체화했습니다.

### 2.1. 백엔드 (BE) 파트

*   **백엔드 A (팀장 / DB·인프라·코어 로직 담당)**
    *   데이터베이스 ERD 심화 설계: 단순 테이블 정의를 넘어 기자재의 '상태(대여 가능, 대여 중, 수리 중, 분실 등)' 관리 및 '이력' 보존 전략 수립.
    *   대여/반납 핵심 비즈니스 로직 및 API 개발.
    *   동시성 제어 로직 구현: Prisma 트랜잭션을 활용한 비관적/낙관적 락(Lock) 전략 적용 (동일 기자재 중복 대여 방지).
    *   서버 배포(AWS EC2 등) 및 CI/CD 파이프라인(GitHub Actions) 구축.
    *   보안: JWT 기반 인증 및 관리자 권한(RBAC)에 따른 API 접근 제어 로직 구현.

*   **백엔드 B (API·기능·통계 담당)**
    *   기자재별 QR 코드 생성 로직 개발 (외부 라이브러리 활용).
    *   관리자용 재고 관리 및 통계 대시보드 API 개발 (복잡한 통계 쿼리 작성).
    *   반납 예정일 도래 시 자동 알림(CronJob 등) 스케줄러 구현.
    *   Swagger(OpenAPI)를 활용한 REST API 명세서 자동화 및 유지보수.

### 2.2. 프론트엔드 (FE) 파트

*   **프론트엔드 A (학생/사용자 뷰·핵심 기능 담당)**
    *   React 초기 프로젝트 환경 세팅 및 공통 컴포넌트(버튼, 모달 등) 개발.
    *   학생용 메인 페이지, 기자재 목록 조회 및 검색 UI 구현.
    *   모바일/웹 브라우저 카메라 연동 및 QR 스캐너 기능 구현 (핵심 기능).
    *   대여 신청 및 반납 처리 클라이언트 로직 연동 (React Query 활용).
    *   PWA(Progressive Web App) 설정 적용 (모바일 접근성 향상).

*   **프론트엔드 B (관리자 뷰·디자인·통계 담당)**
    *   Figma를 활용한 전체 UI/UX 와이어프레임 및 프로토타입 설계.
    *   관리자 전용 대시보드 UI 구현 (재고 현황, 대여 이력 표, 차트 시각화 등).
    *   신규 기자재 등록, QR 코드 출력 페이지 구현.
    *   반응형 웹 디자인 적용 및 CSS 스타일링 (Tailwind CSS 등 활용).

## 3. 주차별 세부 진행 일정 (4주)

기존 일정을 바탕으로, 각 주차별 목표를 명확히 하고 테스트 및 문서화 단계를 강화했습니다.

### 3.1. 1주차: 요구사항 분석 및 기반 설계

*   **공통:**
    *   요구사항 분석 회의 및 기능 정의서 확정.
    *   GitHub 레포지토리 생성, 브랜치 전략(Git Flow 등) 수립 및 컨벤션 정의.
*   **백엔드:**
    *   데이터베이스 ERD 심화 설계 (사용자, 기자재, 대여 이력, 상태 코드 테이블 등).
    *   Prisma 초기 세팅 및 DB 마이그레이션 테스트.
    *   API 엔드포인트 설계 및 Swagger 초기 세팅 (Mock API 제공).
*   **프론트엔드:**
    *   Figma를 이용한 화면 와이어프레임 및 프로토타입 완성.
    *   React 프로젝트 셋업 (라우팅, 전역 상태 관리, React Query 세팅).

### 3.2. 2주차: 핵심 비즈니스 로직 및 UI 컴포넌트 개발

*   **백엔드:**
    *   (BE-A) 사용자 인증/인가, 대여/반납 CRUD API 개발 및 동시성 제어 로직 초안 작성.
    *   (BE-B) QR 코드 생성 API 및 기자재 등록/관리 CRUD API 개발.
*   **프론트엔드:**
    *   (FE-A) 카메라 API 연동 테스트 및 QR 스캔 페이지 레이아웃 구현.
    *   (FE-B) 관리자 페이지 레이아웃, 기자재 목록 테이블 및 공통 UI 컴포넌트 구현.
*   **공통:**
    *   주말 전 백엔드 API 1차 배포 (개발 서버) 및 프론트엔드 연동 테스트 준비.

### 3.3. 3주차: 프론트-백 연동 및 부가 기능 통합

*   **공통:**
    *   구현된 프론트엔드 UI에 백엔드 실제 API 연동 시작.
*   **백엔드:**
    *   (BE-B) 반납 기한 알림 스케줄러 로직 구현 및 테스트.
    *   (BE-A) 프론트엔드 연동 중 발생하는 버그 픽스, 쿼리 최적화 및 보안 로직 강화.
*   **프론트엔드:**
    *   (FE-A) 실제 스캔한 QR 데이터를 API로 전송하여 대여 성공/실패 UI 연동 및 예외 처리.
    *   (FE-B) 관리자 대시보드 통계 데이터 연동 및 QR 코드 이미지 다운로드/다중 출력 기능 구현.

### 3.4. 4주차: 테스트, 최적화 및 프로젝트 산출물 정리

*   **공통 (테스트):**
    *   **동시성 테스트:** 여러 기기에서 동시에 같은 QR을 스캔하여 대여를 시도할 때 DB 무결성 및 락(Lock) 동작 확인.
    *   **크로스 브라우징 및 모바일 테스트:** 실제 스마트폰 브라우저(Safari, Chrome)에서 UI 깨짐 현상 및 카메라 권한 버그 확인.
*   **백엔드:**
    *   최종 서버 배포, 운영 DB 이관 및 CI/CD 파이프라인 점검.
*   **프론트엔드:**
    *   프론트엔드 호스팅 (Vercel, Netlify 등) 최종 배포 및 PWA 동작 확인.
*   **공통 (문서화):**
    *   코드 리팩토링 및 주석 정리.
    *   GitHub README.md 작성 (프로젝트 소개, 실행 방법, 아키텍처 등).
    *   최종 프로젝트 수행 보고서 작성 및 발표 준비.

## 4. 추가 제안 사항 요약

1.  **PWA(Progressive Web App) 도입:** 모바일 기기에서 네이티브 앱과 유사한 사용자 경험을 제공하여 QR 스캔의 접근성을 극대화합니다.
2.  **Swagger(OpenAPI) 활용:** API 명세서를 자동화하여 프론트엔드와 백엔드 간의 소통 비용을 줄이고 개발 효율성을 높입니다.
3.  **CI/CD 파이프라인 구축:** GitHub Actions를 통해 코드 푸시 시 자동 테스트 및 배포가 이루어지도록 하여 안정적인 서비스 운영 환경을 마련합니다.
4.  **상태 관리 라이브러리 최적화:** 서버 데이터 동기화를 위해 React Query를 도입하여 프론트엔드의 상태 관리 복잡도를 낮춥니다.
